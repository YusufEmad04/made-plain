# The Real Customer Pain Test — Scenario Plan v2

A 9:16 reel teaching ONE idea: **a real pain has a number on it. A wish doesn't.**
The viewer should be able to repeat this idea after a single watch.

**Title line:** *The Real Customer Pain Test*
**Duration target:** ~60-65 seconds
**Scene count:** 6 scenes
**Visual rule:** Everything on screen is large enough to read on a phone at arm's length. Minimum body font ≈ 44px. Minimum headline font ≈ 80px.

---

## The single idea

Founders fail because they build for problems that aren't real pains — they're wishes.
The test: when you ask a customer how they handle the problem today, do they give you a **number** (hours, dollars, days, customers lost) — or a vague feeling?
- **Number** → real pain → worth building.
- **No number** → inconvenience → not yet.

That's it. Everything in this reel reinforces that one rule.

## Scene-by-scene

### 01-graveyard — Hook: built it, nobody bought it

**Visual description:**
Three identical product "card" shapes pop in across the upper third of the frame. Each looks like a phone or product mock-up. They sit briefly. Then each one tilts/crumbles in turn — falling apart with a small dust effect. Behind/below them, a giant headline lands:

```
THE PAIN
WASN'T REAL.
```

**On-screen text:**
- Headline (size ≈ 110px): "The pain wasn't real."
- Optional sub (size ≈ 44px): "Most products die for one reason."

**Narration:**
*"Most products fail for one reason. They built something nobody actually needed. Not bad design. Not bad marketing. The pain wasn't real."*

**Sync notes:**
- "Most products fail" → 3 product cards appear
- "for one reason" → cards begin tipping
- "nobody actually needed" → first crumbles
- "Not bad design / marketing" → second / third crumble
- "The pain wasn't real" → headline lands large

**Examples:** the three crumbling product cards (concrete visual stand-ins for failed launches).

### 02-rule — The rule

**Visual description:**
A single very large statement fills the frame, one phrase at a time:

```
A real pain
has a NUMBER
on it.
```

A small "$" mark or number-counter ticks up beside the word "NUMBER" as it lands. The whole scene is one idea, oversized.

**On-screen text:**
- "A real pain" (size 90px) — appears first
- "has a NUMBER" (size 130px, NUMBER in accent) — appears second, big punch
- "on it." (size 90px) — appears last

**Narration:**
*"Here's the test in one line. A real pain has a number on it."*

**Sync notes:**
- "Here's the test in one line" → "A real pain" appears
- "A real pain" → "has a NUMBER" lands big
- "on it" → "on it." lands

**Examples:** The number itself is the example — visible as a counter ticking, making the abstraction physical.

### 03-two-customers — Show the test in action

**Visual description:**
Two characters appear, top half then bottom half. Each has a big quote bubble. Below each quote, a verdict label.

**Top — Customer A:**
- Quote bubble: *"I'd love a better tool."*
- Below: NO NUMBER (in muted/red)
- Below that: INCONVENIENCE

**Bottom — Customer B:**
- Quote bubble: *"I lose 6 hours every Monday."*
- The "6 hours" is circled / highlighted in accent as it appears
- Below: NUMBER FOUND (in green/accent)
- Below that: PAIN

**On-screen text:**
- Quote A (size 56px)
- Quote B (size 56px)
- Verdict labels (size 56px, uppercase)

**Narration:**
*"Listen to two customers. The first says — I'd love a better tool. No number. That's not a pain. That's an inconvenience. The second says — I lose six hours every single Monday. Six hours. There's your number. That's a real pain."*

**Sync notes:**
- "First says" → Customer A quote appears
- "No number / inconvenience" → INCONVENIENCE label
- "Second says" → Customer B quote appears
- "I lose six hours" → "6 hours" highlighted with marker / circle
- "real pain" → PAIN label lands

**Examples:** The two literal customer quotes — the contrast IS the example.

### 04-question — How to find the number

**Visual description:**
Two big question cards stacked. The bad question is shown first, then crossed out. The good question replaces it.

**Card 1 — DON'T ASK:**
*"Would you pay for this?"*
Customer answer below: *"Sure!"* — strikethrough, marked with a red ✗
Sub-label: LIE.

**Card 2 — ASK:**
*"How do you solve it today?"*
Customer answer below: *"3 spreadsheets. 2 hours a day."*
The numbers "3" and "2 hours" are highlighted in accent as they appear.
Sub-label: TRUTH.

**On-screen text:**
- Card titles (size 60px, ✗ / ✓ marker)
- Question text (size 56px)
- Answer text (size 50px, numbers highlighted)
- Verdict label (size 48px)

**Narration:**
*"So how do you find the number? Don't ask — would you pay for this. They'll say yes to be polite. Ask — how do you solve it today. Then listen for the numbers. Time. Money. Workarounds."*

**Sync notes:**
- "Don't ask" → Card 1 appears
- "would you pay for this" → bad question text
- "yes to be polite" → "Sure!" + ✗ + LIE
- "Ask" → Card 2 appears
- "how do you solve it today" → good question text
- "listen for the numbers" → answer text appears, numbers highlighted in accent

**Examples:** "3 spreadsheets" and "2 hours a day" are the literal examples — concrete countable artifacts.

### 05-challenge — Try it this week

**Visual description:**
A clean challenge card with four big lines, each appearing in turn:

```
5 people.
Ask the question.
Listen for numbers.
3 / 5 = SIGNAL.
```

The number "5" in line 1 ticks up from 0→5. The "3 / 5" in line 4 is large, accent color.

**On-screen text:**
- Each line: ≥56px font
- Line 4: 80px font, accent color

**Narration:**
*"Try it this week. Find five people. Ask the question. If three describe the same problem with real numbers, unprompted — you've found something worth building."*

**Sync notes:**
- "five people" → line 1
- "Ask the question" → line 2
- "real numbers" → line 3
- "three describe" → line 4 (big)
- "worth building" → all four lines hold

**Examples:** "5 people" / "3 of 5" are literal numbers that ground the challenge.

## Visual primitives needed

- **SceneShell** — paper background + small "§ THE PAIN TEST" eyebrow only (small footer-style, not a header that competes with content). Everything else is empty space ready for big text.
- **HugeLine** — single very large line of text, configurable size 80-130px, with fade/rise animation.
- **TombstoneCard** — a product-mock card that crumbles (rotate + fall + fade) on cue. Used in Scene 01.
- **QuoteBubble** — character + speech bubble with very large quote text. Used in Scene 03.
- **VerdictLabel** — uppercase tagged label (NUMBER FOUND / NO NUMBER / PAIN / INCONVENIENCE / LIE / TRUTH).
- **QuestionCard** — card with a heading mark (✗/✓), a question, an answer, and a verdict line. Used in Scene 04.
- **ChallengeList** — vertical big-font list with optional number tick. Used in Scene 05.
- **HighlightSpan** — inline span that gets a marker-pen accent stroke when its cue fires. Used to highlight numbers.
